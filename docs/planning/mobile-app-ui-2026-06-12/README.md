# 모바일 앱형 UI 기획 초안

작성일: 2026-06-12

## 배경

Playwright 모바일 뷰포트 기준으로 현재 메인 화면을 확인한 결과, 모바일에서도 데스크톱 웹 랜딩 페이지 구조가 그대로 축소되어 보인다. 목표는 모바일 웹을 단순 반응형 페이지가 아니라, 사용자가 모바일 앱처럼 인식할 수 있는 구조로 재설계하는 것이다.

## 현재 모바일 화면의 문제

1. 상단 프로필, 로그아웃, 설정 버튼이 모바일 첫 화면에서 과하게 드러난다.
2. 대형 랜딩 문구가 첫 화면 대부분을 차지해 실제 핵심 기능인 AI 마음대화 진입이 늦다.
3. CTA가 웹 섹션 내부 버튼처럼 보이며, 앱의 주요 행동으로 고정되어 있지 않다.
4. 서비스 설명 섹션이 모바일 앱의 도움말 또는 온보딩이 아니라 긴 웹 페이지 섹션처럼 보인다.
5. 하단 탐색 구조가 없어 사용자가 현재 위치와 주요 기능을 앱처럼 이해하기 어렵다.

## 화면 정의 방향

### 1. 홈 / 오늘의 대화 진입

- 상단은 모바일 앱바로 축소한다.
- 첫 화면은 대형 히어로 문구 대신 오늘의 AI 마음대화 시작 카드로 구성한다.
- 사용자가 바로 한 문장을 입력하거나, 상황 카드에서 대화 시작점을 고를 수 있게 한다.
- 하단에는 홈, 대화, 기록, 나 탭을 둔다.

### 2. AI 마음대화 / 채팅 앱처럼

- 대화 화면은 웹 카드가 아니라 실제 채팅방 구조로 만든다.
- 입력 composer는 화면 하단에 고정한다.
- 말풍선은 좌우 흐름과 간격을 유지해 실제 대화처럼 보이게 한다.

### 3. 서비스 설명 / 앱 온보딩 카드

- 긴 설명 섹션을 줄이고, 앱 내부 도움말처럼 짧은 단계 카드로 재구성한다.
- 설명은 “꺼내기 → 나누기 → 정리하기 → 멈추기” 흐름으로 보여준다.
- CTA는 하단 고정 버튼으로 유지해 사용자가 설명을 읽은 뒤 바로 시작할 수 있게 한다.
